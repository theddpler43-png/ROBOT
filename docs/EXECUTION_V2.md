# Execution Ratio V2

## Цель

Execution Ratio является главным фильтром ROBOT Browser V9.

Его задача — определить, насколько характер исполнения сделок похож на работу маркет-мейкера.

Execution V2 состоит из трёх независимых компонентов.

---

# 1. Price Match

Вес:

50%

Проверяет:

Исполняются ли сделки по уровням, присутствующим в стакане.

Высокая оценка:

- сделки совпадают с Bid/Ask;
- сделки проходят по ближайшим уровням.

Низкая оценка:

- сделки происходят далеко от видимого стакана.

---

# 2. Level Concentration

Вес:

30%

Проверяет:

Повторяются ли сделки на одних и тех же ценовых уровнях.

Пример хорошего рынка:

100.00
100.01
100.00
100.01
100.00

Пример плохого рынка:

100.00
100.08
99.95
100.21
100.02

---

# 3. Execution Rhythm

Вес:

20%

Проверяет:

Есть ли повторяющийся ритм исполнения.

Например:

Bid
Ask
Bid
Ask

или

Bid
Bid
Bid
Bid

Хаотичная последовательность уменьшает оценку.

---

# Формула

Execution =
PriceMatch × 0.50 +
LevelConcentration × 0.30 +
ExecutionRhythm × 0.20

---

# Требования

Каждый компонент должен быть реализован отдельной функцией.

calc_execution_pct() остаётся единственной публичной функцией.

Внутренние функции:

- _price_match_score()
- _level_concentration_score()
- _execution_rhythm_score()

Это позволит независимо улучшать каждый компонент без изменения интерфейса.
